# WindowsTools

## V1.1 开发方案

目标：在现有 V1 基础上完善列表查找体验，不做复杂分类、不引入配置系统、不扩展到软件管理器。重点解决：筛选可用、排序直观、路径可按盘符过滤、快捷方式和真实程序有明确区分。

### 一、列表字段

列表保持表格形态，建议字段调整为：

| 字段 | 说明 |
| --- | --- |
| 软件 | 软件显示名称。点击表头时按名称升序/降序切换排序。 |
| 类型 | 显示 `真实程序`、`快捷方式`、`无效快捷方式`。用于明确区分当前项来源形态。 |
| 来源 | 显示 `系统开始菜单`、`用户开始菜单`、`公共桌面`、`用户桌面`。 |
| 路径 | 优先显示真实启动目标路径；无法解析时显示快捷方式路径。 |

类型区分采用“类型列 + 轻微行色辅助”的方案：

- `真实程序`：普通行样式。
- `快捷方式`：类型列显示快捷方式，行色可保持普通，不强依赖颜色。
- `无效快捷方式`：类型列显示无效快捷方式，可使用浅红/浅灰行色提示，但必须同时有文字说明。

说明：不只用颜色区分，避免用户看不清，也符合主流桌面表格的可读性做法。

### 二、筛选设计

顶部工具栏建议为：

```text
搜索：[ 输入软件名 / 路径 / 来源 ]  来源：[ 全部 ▼ ]  盘符：[ 全部 ▼ ]  类型：[ 全部 ▼ ]  [清空] [刷新] [管理员启动]
```

筛选只保留 3 个轻量条件：

1. 搜索框
   - 实时筛选，不需要点击搜索按钮。
   - 搜索范围：软件名、来源、路径。
   - 不区分大小写。
   - 支持空格多关键词，所有关键词都匹配才显示。
   - 按 `Esc` 或点击 `清空` 恢复。

2. 来源筛选
   - 选项：`全部`、`开始菜单`、`桌面`。
   - `开始菜单` 包含：系统开始菜单、用户开始菜单。
   - `桌面` 包含：公共桌面、用户桌面。

3. 盘符筛选
   - 选项根据当前扫描结果动态生成，例如：`全部`、`C:`、`D:`、`E:`。
   - 盘符来源优先使用真实启动目标路径。
   - 如果快捷方式无法解析真实目标，则使用快捷方式自身路径参与筛选。

4. 类型筛选
   - 选项：`全部`、`真实程序`、`快捷方式`、`无效快捷方式`。
   - 这是为了明确区分快捷方式和真实软件，不做主观软件分类。

说明：这里的“类型”不是软件用途分类，不会出现浏览器、办公、游戏这类规则，避免误判和复杂化。

### 三、排序设计

只做表头点击排序，不增加排序下拉框。

- 点击 `软件` 表头：按软件名 A-Z / Z-A 切换。
- 点击 `路径` 表头：按路径 A-Z / Z-A 切换。
- 默认排序：软件名升序。
- 表头文字可显示方向提示，例如：`软件 ↑`、`软件 ↓`。

当前阶段只要求 `软件` 必须支持点击排序；`路径` 排序可以和盘符筛选一起实现，成本低则同步完成。

### 四、启动逻辑

启动逻辑保持 V1 当前方向：

- `.exe`：直接按真实程序处理。
- `.lnk`：先解析真实目标，再对真实目标执行管理员启动。
- 无法解析 `.lnk`：标记为 `无效快捷方式`，点击启动时给出明确提示。

失败提示必须包含：

- 软件名
- 当前显示路径
- 失败原因，例如：目标不存在、没有目标程序、目标不是 exe

### 五、验收标准

- 右侧滚动条可见、可拖动，鼠标滚轮可滚动。
- 搜索 `code` 能过滤列表；搜索多个关键词也能正常过滤。
- 来源筛选能在 `全部 / 开始菜单 / 桌面` 间切换。
- 盘符筛选能按 `C:`、`D:`、`E:` 等实际存在盘符过滤。
- 类型筛选能区分 `真实程序`、`快捷方式`、`无效快捷方式`。
- 点击 `软件` 表头能在升序和降序之间切换。
- 点击无效快捷方式时，不再显示笼统错误，而是显示明确失败原因。
