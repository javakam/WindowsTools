# WindowsTools

## V1.1 开发方案

目标：在现有 V1 基础上保留搜索栏，增加“来源筛选”，并扫描开始菜单已固定快捷方式。不做复杂分类、不做盘符筛选、不做类型筛选、不做表头排序、不扩展成软件管理器。

### 一、界面方案

顶部工具栏保留最少控件：

```text
搜索：[ 输入软件名 / 来源 / 路径 ]  来源：● 全部  ○ 开始菜单  ○ 开始固定  ○ 桌面
```

列表字段保持现有三列：

| 字段 | 说明 |
| --- | --- |
| 软件 | 软件显示名称。 |
| 来源 | 显示真实扫描来源：系统开始菜单、用户开始菜单、公共桌面、用户桌面、开始固定。 |
| 路径 | 显示当前扫描到的快捷方式或程序路径。 |

列表启动规则：

- `.exe` 程序直接以管理员权限启动。
- 指向 `.exe` 的 `.lnk` 快捷方式先解析真实程序，再以管理员权限启动。
- 指向 `.md`、`.txt`、`.pdf` 等文档的快捷方式仍然显示；启动时通过 Windows 默认关联程序打开，并让关联程序以管理员权限运行。
- 如果某类文档没有默认打开程序，才显示明确错误。

### 二、搜索与来源筛选

搜索栏规则：

- 保留搜索框，默认显示全部。
- 输入时立即筛选，不需要点击搜索按钮。
- 搜索范围：软件名、来源、路径。
- 支持空格分隔的多个关键词，所有关键词都匹配才显示。
- 按 `Esc` 清空搜索内容，并恢复当前来源范围下的列表。

来源筛选平铺显示四个单选项：

| 选项 | 匹配范围 |
| --- | --- |
| 全部 | 显示所有扫描结果。 |
| 开始菜单 | 显示系统开始菜单、用户开始菜单。 |
| 开始固定 | 显示开始菜单已固定快捷方式。 |
| 桌面 | 显示公共桌面、用户桌面。 |

筛选规则：

- 默认选中 `全部`。
- 切换来源后立即更新列表显示。
- 搜索和来源筛选同时生效。
- 不增加盘符筛选、不增加类型筛选、不增加排序下拉。
- 列表仍按软件名称升序显示，保持简单稳定。

### 三、启动逻辑

保持现有管理员启动逻辑：

- 管理员启动入口放在列表项右键菜单中。
- `.lnk` 快捷方式先解析真实目标，再以管理员权限启动目标程序。
- `.exe` 直接以管理员权限启动。
- `.md`、`.txt`、`.pdf` 等文档目标通过默认关联程序以管理员权限打开。
- 无法解析或目标不存在时，显示明确失败原因。

### 四、验收标准

- 右侧滚动条可见、可拖动。
- 搜索栏存在，输入关键词能实时筛选软件名、来源、路径。
- `全部` 能显示所有扫描结果。
- `开始菜单` 只显示系统开始菜单、用户开始菜单来源。
- `桌面` 只显示公共桌面、用户桌面来源。
- `开始固定` 只显示开始菜单已固定快捷方式来源。
- 右键列表项选择 `管理员启动` 能触发 UAC。
- 无法启动的软件显示明确失败原因。
