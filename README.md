# WindowsTools

## V1.1 开发方案

目标：在现有 V1 基础上保留搜索栏，并只增加“来源筛选”。不做复杂分类、不做盘符筛选、不做类型筛选、不做表头排序、不扩展成软件管理器。

### 一、界面方案

顶部工具栏保留最少控件：

```text
搜索：[ 输入软件名 / 来源 / 路径 ]  来源：[ 全部 ▼ ]  [刷新] [管理员启动]
```

列表字段保持现有三列：

| 字段 | 说明 |
| --- | --- |
| 软件 | 软件显示名称。 |
| 来源 | 显示真实扫描来源：系统开始菜单、用户开始菜单、公共桌面、用户桌面。 |
| 路径 | 显示当前扫描到的快捷方式或程序路径。 |

### 二、搜索与来源筛选

搜索栏规则：

- 保留搜索框，默认显示全部。
- 输入时立即筛选，不需要点击搜索按钮。
- 搜索范围：软件名、来源、路径。
- 支持空格分隔的多个关键词，所有关键词都匹配才显示。
- 按 `Esc` 清空搜索内容并恢复当前来源范围下的列表。

来源下拉只保留三个选项：

| 选项 | 匹配范围 |
| --- | --- |
| 全部 | 显示所有扫描结果。 |
| 开始菜单 | 显示系统开始菜单、用户开始菜单。 |
| 桌面 | 显示公共桌面、用户桌面。 |

筛选规则：

- 默认选中 `全部`。
- 切换来源后立即刷新列表显示。
- 搜索和来源筛选同时生效。
- 不增加盘符筛选、不增加类型筛选、不增加排序下拉。
- 列表仍按软件名称升序显示，保持简单稳定。

### 三、启动逻辑

保持现有管理员启动逻辑：

- `.lnk` 快捷方式先解析真实目标，再以管理员权限启动目标程序。
- `.exe` 直接以管理员权限启动。
- 无法解析或目标不存在时，显示明确失败原因。

### 四、验收标准

- 右侧滚动条可见、可拖动。
- 搜索栏存在，输入关键词能实时筛选软件名、来源、路径。
- `全部` 能显示所有扫描结果。
- `开始菜单` 只显示系统开始菜单、用户开始菜单来源。
- `桌面` 只显示公共桌面、用户桌面来源。
- 选择软件后点击 `管理员启动` 能触发 UAC。
- 无法启动的软件显示明确失败原因。
