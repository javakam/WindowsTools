# AGENTS.md

## 项目定位

这是一个 Windows 软件启动小工具，使用 Python + Tkinter 开发，通过扫描开始菜单和桌面快捷方式，支持搜索、来源筛选和管理员权限启动。

当前目标是保持工具简单、稳定、轻量，不扩展成复杂的软件管理器。

## 开发原则

- 使用 Python 标准库、Tkinter、pywin32 相关能力优先。
- 不引入 Web 框架、前端框架、数据库、后台服务等重型依赖。
- 不做过度设计，不拆分过多文件，除非功能复杂度明显上升。
- 修改应尽量集中在当前相关功能，避免无关重构。
- 保持界面简洁，优先保证 Windows 下可用性和性能。

## 功能边界

当前工具只关注：

- 扫描系统开始菜单、用户开始菜单、公共桌面、用户桌面。
- 展示软件名称、来源、路径。
- 支持搜索和来源筛选。
- 支持以管理员权限启动 `.exe`。
- 支持解析 `.lnk` 快捷方式。
- 对文档类目标，通过 Windows 默认关联程序启动。

除非明确要求，不要增加：

- 软件卸载/安装管理。
- 注册表深度扫描。
- 进程管理。
- 自动更新。
- 账号系统。
- 复杂分类、标签、排序、插件系统。

## Windows 兼容性注意

- 快捷方式解析应使用 `pywin32` / COM 能力。
- 管理员启动应使用 Windows `ShellExecuteW` 的 `runas`。
- 文档类文件启动应尊重 Windows 默认关联程序。
- 错误提示要明确说明失败原因。
- 不要假设所有快捷方式目标都存在。
- 不要假设所有文件类型都有默认打开程序。

## UI 要求

- 使用 Tkinter / ttk 保持原生轻量风格。
- 控件数量保持克制。
- 搜索输入应即时筛选。
- 列表滚动条应可见、可拖动。
- 状态栏应反馈扫描结果、筛选结果和启动结果。

## 验证要求

修改后至少检查：

- 程序能正常启动。
- 搜索功能可用。
- 来源筛选可用。
- 列表可滚动。
- 双击或点击“管理员启动”能触发启动逻辑。
- 启动失败时有明确错误提示。

如修改打包相关内容，还需要确认 PyInstaller spec 中的 hiddenimports 是否仍包含：

- `pythoncom`
- `pywintypes`
- `win32com.client`
